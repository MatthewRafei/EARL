Your First Program

To jump right into testing out EARL, do the following to implement a Hello World! program.

Create a new file called hello.earl 

print("Hello World!");

Then run 

earl ./hello.earl

Output: 

Hello World!

Variable Declarations

let <id> = expr;

Mutation

<id> | expr = expr;

Conditionals

if expr { [stmt] }

else if expr { [stmt] }

else { [stmt] }

Pattern Matching

Pattern matching allows you to match on some pattern. This is the case for all primitives int, str, list etc.

A match consists of a two things.

  1. The pattern → this comes right after the match keyword
  2. Branches → these are all the different patterns can be

Each branch can either specify one pattern or multiple patterns with the | (or) operator. The when keyword acts as a guard for the branch. If the pattern matches, only then is the guard evaluated. If the guard is true, only then will the branch activate.

And finally, the base case consists of the pattern _. If none of the other branches above this one happens, this one is guaranteed to happen (unless there is a guard).

Grammar TODO

Examples

let x = 4;
let msg = "foo";

match x {
    1 -> { print("x is 1"); }
    2 | 3 -> { print("x is two or three"); }
    [1,2,3] when type(x) == "list" -> { print(x is a list and is [1,2,3]") };
    4 when msg = "bar" -> { print("x is 4 with msg = bar"); }
    4 when msg = "foo" -> { print("x is 4 with msg = foo"); }
    _ when msg = "hello" -> { print("base case, 4 is something else and msg = hello"); }
    _ -> { print("base case, 4 is something else"); }
}

Output: 

x is 4 with msg = foo

let x = some(4);

match x {
    none -> {
        print("x is none");
    }
    some(3) -> {
        print("x is some 3");
    }
    some(k) when k % 2 == 1 -> {
        print("x is something and is odd");
    }
    some(k) -> {
        # This branch will execute.
        # `k` is the unwrapped value of `x`.
        print("x is: ", k);
        print(x.unwrap() == k);
    }
    some(_) -> {
        print("x is something");
    }
    _ -> {
        print("x is something else");
    }
}

Output: 

x is: 4
true

While Loops

while expr { [stmt] }

For Loops

As of now, the for loop only enumerates by incrementing by 1. It takes a start (\(s\)) and an end (\(e\)) and will not end until \(s = e\).

\[ \sum_{i = start}^{i < end} i = i + 1 \]

You can modify i within the body of the loop.

for <id> in expr .. expr { [stmt] }

Function Definitions

This is how to create a function. The @world, @pub and @ref are attributes. See Attributes for a description on them.

Grammar

*( @ [ pub | world ])

fn <id> ( [ * ( @ref ) <id> ] ) { [stmt] }

Examples

@world @pub
fn func1(@ref x, y, @ref z) {
    ...
}

fn func2() {
    ...
}

Statement Expressions

Statement expressions are statements that are only evaluated for their side effects such as printing, file IO, or in-place mutation such as List.rev.

Grammar

<id> *( ( expr ) | [ expr ] | . <ident> ) ;

Examples

print("hello world");
my_list.rev();

Imports

EARL looks for the files relative to the directory that it was invoked in. During runtime, if an import statement is hit, it will immediately interpret that file. If there is no processing in the @world scope, only the function definitions are gathered.

Grammar

import "filepath"

Examples

import "std/list.earl"
import "std/math.earl"
import "std/utils.earl"

Modules

Modules allow other files to access resources to the file that has the module declared in it. If a file does not have it, the entire file is private. While modules do allow other files to access resources, only resources that have the @pub attribute.

It is expected to have the module declaration at the top of the file. If you do not follow this rule, undefined behavior will ocurr.

Grammar

mod <id>

Examples

file: my-math.earl 

mod MyMath

fn sum(a, b) { return a+b; }

file: main.earl 

mod Main

import "my-math.earl"

let s = MyMath.sum(1, 2);

Attributes

Attributes embeds meta information into variables and functions. You can have none, one, or multiple.

Name Vars Funcs Desc
@pub UNIMPLEMENTED YES Makes the function available to other files
@world NO YES The function closes in the world scope
@ref YES NO Declares the var to be a reference

NOTE: @ref can be used as a function parameter and that function will take a reference to the value passed to it.

Grammar

@ *(pub | world | ref)

Examples

@world @pub
fn test_world() {
    # `i` will print since it is in the world scope.
    # The function is also marked as public so other
    # modules can have access to this function.
    print(i)
}

@pub
fn take_ref(@ref ptr) {
    ptr = ptr + 1;
}

let i = 1;
test_world();

@ref let ptr = i;
ptr = ptr + 1;

assert(ptr == 2 && i == 2);

take_ref(i);

assert(i == 3);

Classes

While classes are available, support for inheritance is not yet implemented.

Grammar

@ *(pub | world | ref)

class <id> ?([ <arg1>, <arg2>,…, <argN> ]) {

@ *(pub | world | ref) <member1> = <id> | expr ;

@ *(pub | world | ref) <member2> = <id> | expr ;

<methods>

}

Examples

# This class takes no arguments for the
# constructors and is also not public,
# so other modules cannot use this class.

class NoPassesArgs {
    x = 1;
    y = 2;

    @pub fn print() {
        The use of `this` keyword is optional
        print(this.x, y);
    }
}

@pub class Point [x, y, name] {
    x = x;
    y = y;
    name = name;

    # Because this is public, you can access
    # it directly from outside the class.
    @pub desc = "A point class"

    @pub set_x(newx) {
        this.x = newx
    }

    @pub dump() {
        print(this.x);
        print(this.y);
        print(this.name);
        print(this.desc);
    }
}

@pub class Stack [init] {
    # You could just assign `init` to `lst` here,
    # but for this example, this is done in the constructor.
    lst = [];
    sz = len(lst);

    # The constructor gets called
    # right after class instantiation.
    fn constructor() {
        for i in 0..len(init) {
            this.push(init[i]);
        }
    }

    @pub fn push(x) {
        this.lst.append(x);
        this.sz = this.sz+1;
    }

    @pub fn pop() {
        this.lst.pop(this.sz-1);
        this.sz = this.sz-1;
    }
}

Datatypes

The following is a list of the currently implemented primitive types.

int

Your basic 32 bit integer datatype.

Note: They do over/under-flow

bool

The boolean datatype. Can either be true or false.

char

A single character. They are surrounded by single quotes '.

str

A string is a list of char's. They are surrounded with double quotes ".

list

A list is a growable array. They can hold any datatype, and you can even mix multiple datatypes in a single list.

option

Because all variables need to be initialized when using a `let` statement, it may be annoying to be forced to give it some initial value. The none type allows you to set a variable to it and can be reassigned later.

To assign a value to a none-type, you must wrap it in some i.e.,

let x = none;

x = some(9);

Note: A lot of the functionality of the language will break if you use none as another datatype before giving it an actual value.

closure

Closures are anonymous functions that close in the outer scope. They can be used as functions or passed around as variables.

Examples

fn take_sum_closure(f, a, b) {
    f(a, b);
}

let msg = "hello world";
let hello = |_| {
    print(msg);
};

let sum = |a, b| {
    print("in sum");
    return a+b;
};

hello();

print(take_sum_closure(sum, 1, 2));

Output: 

hello world
in sum
3

file

The file type is a file handler. It handles opening, closing, reading, and writing to files.

Note: It is up to the user to close() the file handle.

unit

Currently not being used in EARL, only in the interpreter. This type gets returned when doing things such as side effects, non-returning function calls, etc.

Intrinsics

Intrinsics are functions that are automatically built into the interpreter. You do not need to import anything to use these functions.

print

print(arg1: any, arg2: any, ..., argN: any) -> unit

Will print all elements if it can. Adds a newline at the end and flushes the output.

assert

assert(arg1: any, arg2: any, ..., argN: any) -> unit

Tests each argument for 1. If any of the conditions fails, the program immediately crashes.

len

len(arg: list|str) -> int

Expects either a list or string. Will give the length as an integer.

some(arg: any) -> option<any>

Wraps arg in an option type.

type(arg: any) -> str

Returns the type of arg as a str.

open(fp: str, mode: str) -> file

Opens the filepath fp with mode mode. Mode must either be r for read or w for write.

unimplemented(arg1, arg2, ..., argN)

Prints arg1..arg\(N\) to stderr and exits with non-zero exit code.

Member Intrinsics

Member intrinsics are the same as intrinsics, except they are member functions (or methods) that types have built into them. You access these with the dot `.` notation i.e.,

let lst = [1,2,3];

lst.rev();

list Implementes

rev() -> unit

Reverses the list inplace.

append(arg1: any, arg2: any, ..., argN: any) -> unit

Will append arg1..arg\(N\) to the list.

pop(idx: int) -> unit

Will remove the element at index idx.

filter(cl: closure) -> list

Takes the closure cl and creates a new list of all the elements where cl returns true.

str Implements

split(delim: str) -> list

Split a string by the delimiter delim.

pop(idx: int) -> unit

Will remove the element at index idx.

char Implements

ascii() -> int

Returns the ascii code of the character.

option Implements

is_none() -> bool

Returns true if the value is none, false otherwise.

is_some() -> bool

Returns true if the value is some, false otherwise.

file Implements

close() -> unit

Closes an opened file.

read() -> str

Get the contents of a file as a str.

dump() -> unit

Print the contents of a file.

StdLib

Upon installing EARL, the standard library is installed into /usr/local/include/EARL/std. The first place that EARL looks when evaluating an import statement is /usr/local/include/EARL/. This means that all you need to do to import these files is do: import "std/<file>.earl".
